You don't need to know everything on day one. This is a reference to come back to as you work across different projects.

## Kubernetes

- [Kubernetes Documentation](https://kubernetes.io/docs/home/) - the official docs, genuinely good
- [Kubernetes the Hard Way](https://github.com/kelseyhightower/kubernetes-the-hard-way) - build a cluster from scratch to understand what's actually happening
- [learnk8s](https://learnk8s.io/) - practical articles and visual guides
- [CKAD/CKA Exam Prep](https://github.com/dgkanatsios/CKAD-exercises) - hands-on exercises even if you're not taking the exam

## AWS

- [AWS Well-Architected Framework](https://aws.amazon.com/architecture/well-architected/) - the mental model for good AWS architecture
- [AWS Documentation](https://docs.aws.amazon.com/) - comprehensive, use the search
- [Last Week in AWS](https://www.lastweekinaws.com/) - newsletter covering AWS changes with commentary
- [AWS Skill Builder](https://skillbuilder.aws/) - free courses from AWS

## Terraform

- [Terraform Documentation](https://developer.hashicorp.com/terraform/docs) - official docs
- [Terraform Best Practices](https://www.terraform-best-practices.com/) - community-driven guide
- [Terraform Up & Running](https://www.terraformupandrunning.com/) - the book most people start with

## Pulumi

- [Pulumi Documentation](https://www.pulumi.com/docs/) - official docs for all languages
- [Pulumi Examples](https://github.com/pulumi/examples) - real-world examples in TypeScript and Python

## Python

- [Real Python](https://realpython.com/) - practical tutorials and guides
- [Python Documentation](https://docs.python.org/3/) - official docs
- [Ruff Documentation](https://docs.astral.sh/ruff/) - our linter/formatter of choice
- [uv Documentation](https://docs.astral.sh/uv/) - our package manager

## TypeScript

- [TypeScript Handbook](https://www.typescriptlang.org/docs/handbook/) - official guide
- [Total TypeScript](https://www.totaltypescript.com/) - advanced TypeScript patterns
- [Bun Documentation](https://bun.sh/docs) - our JS/TS runtime

## Rust

- [The Rust Book](https://doc.rust-lang.org/book/) - start here
- [Rust by Example](https://doc.rust-lang.org/rust-by-example/) - learn by doing
- [Rustlings](https://github.com/rust-lang/rustlings) - small exercises to get comfortable

## Go

- [Go Documentation](https://go.dev/doc/) - official docs
- [Go by Example](https://gobyexample.com/) - practical examples
- [Effective Go](https://go.dev/doc/effective_go) - idiomatic Go patterns

## Docker & Containers

- [Docker Documentation](https://docs.docker.com/) - official docs
- [Dockerfile Best Practices](https://docs.docker.com/build/building/best-practices/) - write better images
- [Dive](https://github.com/wagoodman/dive) - inspect image layers (we have this installed)

## DevOps & SRE

- [Google SRE Book](https://sre.google/sre-book/table-of-contents/) - free online, foundational reading
- [The Phoenix Project](https://itrevolution.com/product/the-phoenix-project/) - the novel that explains DevOps
- [DORA Metrics](https://dora.dev/) - how to measure engineering performance

## Security

- [OWASP Top 10](https://owasp.org/www-project-top-ten/) - the most critical web security risks
- [AWS Security Best Practices](https://docs.aws.amazon.com/IAM/latest/UserGuide/best-practices.html) - IAM and access control

## Networking

- [Cloudflare Learning Center](https://www.cloudflare.com/learning/) - great explanations of DNS, TLS, HTTP, and more
- [How DNS Works](https://howdns.works/) - visual comic explaining DNS

## Staying Current

- [TLDR Newsletter](https://tldr.tech/) - daily tech news digest
- [Changelog](https://changelog.com/) - podcasts covering open source and software
- [Hacker News](https://news.ycombinator.com/) - community discussion (use with moderation)

## Internal Knowledge

- Ask in the team chat. Seriously. Someone has probably solved your problem before.
- Check project READMEs and `docs/` folders for project-specific knowledge.
- Pair with someone who's worked on the project before. 30 minutes of pairing beats hours of solo debugging.
